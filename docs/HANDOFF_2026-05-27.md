## Known Risks / Watch Items
1. **Model variance on tool calls.** Three WSS smoke runs, three different tool-call patterns. Same prompt. The model is non-deterministic about *when* to call `scale_step` and `query_evidence`. Sometimes within 1 turn of Localizer push, sometimes never in 7 turns. This is expected behavior of `gpt-realtime-2` and only browser voice testing across many sessions will fully characterize it.
2. **Voice-side untested.** WSS smoke proves server-side correctness. Audio routing, mic permission, semantic_vad pacing under real voice, transcript saving on session end are unverified post-merge.
3. **OPENAI_API_KEY in Amplify env vs Secrets Manager** тАФ `getOpenAIKey()` in `src/lib/db-query.ts` reads from `app_settings` table; the historian session route uses `getOpenAIKey` from `src/lib/secrets.ts` which reads `process.env.OPENAI_API_KEY`. Two code paths read the OpenAI key from different sources тАФ confusing but not currently broken.
4. **Scale `responses`/`raw_score` schema drift** тАФ these columns were NOT NULL but the legacy submit handler didn't populate them. Migration 047 relaxed them; new `scale_step` populates both pairs (responses+raw_responses, total_score+raw_score). If anyone touches the scoring code, mirror updates need to stay consistent.
5. **Variance test artifact** тАФ `qa/historian-wss-smoke.py` is non-deterministic by design (real model calls). Reading its output requires interpretation; don't treat 0 tool calls as a failure unless the prompt clearly should have triggered them.
