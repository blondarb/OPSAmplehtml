# Triage Classifier Bake-off — Sonnet 4.6 vs Sonnet 5 (2026-07-12)

**Context.** First validation baseline on the CORRECTED 26-case set (post
scramble-remediation; label↔document pairing verified), against the merged
safety architecture (PR #158: scoring floors + emergency gateway + strict
model-output validation), scorer prompt `neurology-outpatient-scorer-v2026-07-12`.
Harness: `scripts/run-validation.ts` (4 runs per case per model: 1 baseline +
3 repeats; majority tier per case). Comparison: `scripts/compare-validation-models.ts`.
Independent cross-family check of disagreements: `scripts/grade-validation-disagreements.ts`
(gpt-oss-120b on Bedrock, neutral document-only prompt).

**Models.** `us.anthropic.claude-sonnet-4-6` (current prod classifier) vs
`us.anthropic.claude-sonnet-5` (candidate: cheaper $2/$10 vs $3/$15, BAA-covered;
NOTE: rejects the `temperature` parameter — repeats measure default-sampling
variance for Sonnet 5 vs t=0/0.2 spread for Sonnet 4.6).

## Findings found by running this baseline (fixed before the scored run)

1. **Governed-vocabulary mismatch killed legitimate redirects.** Kim
   (vertigo + hearing loss) failed 4/4: model output `"ENT"`, validator
   required exact `"ENT / Otolaryngology"`, prompt never enumerated the
   governed values. Fix: `resolveNonNeuroSpecialty()` normalizes spelling
   variants into governed values (never widens the set; unresolvable still
   fails closed) + prompt now enumerates the vocabulary (31b904a).
2. **Sonnet 5 rejects `temperature`** ("deprecated for this model") — all 104
   calls failed until the runner omitted it for 5-family models (31b904a).

## Independent grading of disagreements

Independent model: `openai.gpt-oss-120b-1:0` (Bedrock, neutral document-only
prompt, no expected tier or Claude answer shown). 14 disagreement cases graded.

Pattern: on most over-triage cases the independent grader lands BETWEEN the
expected label and Claude (semi_urgent where expected=routine_priority,
Claude=urgent) — i.e., disagreements live in the calibration band, not the
safety-miss band. On several (delgado, patel->urgent-not-emergent) it sides
with urgency over the label.

**Label-quality flags for clinical sign-off (Steve/Prachi):**
- **jennings**: expected routine_priority, but BOTH Claude (urgent 4/4) and
  the independent grader (emergent, "recurrent crescendo TIA, high ABCD2")
  read urgent TIA content in the document. Label or document needs review.
- **kim**: expected semi_urgent (card theme: gentamicin ototoxicity), but the
  paired document reads as RESOLVED BPPV seeking reassurance — independent
  grader says routine_priority, Claude says routine. The one nominal
  UNDER-triage in the run is probably a label/document pairing question, not
  a scorer miss. (Also note: sudden hearing loss, if actually present, is
  time-critical — needs the clinical eye.)
- **Grader is not gospel either**: it called jimenez (first seizure + frontal
  mass with midline shift) semi_urgent — an under-call by any standard.
  Claude's emergent is defensible there.
